# ADR-001: Use Harness Engineering Scaffold

**Status**: accepted

**Date**: 2026-05-22

## Context

We need a standardized project structure that supports AI-assisted development. The project must include guardrails (security checks, quality gates), behavioral rules for AI coding agents, and a knowledge base for architecture decisions.

## Decision

Use the Harness Engineering Copier template as the project scaffold. This provides:

- CLAUDE.md as the AI agent entry point
- Security guard hooks (PreToolUse + Stop + PostToolUse)
- Modular `.claude/rules/` for behavioral guidelines
- `docs/decisions/` for Architecture Decision Records

## Consequences

**Positive:**
- Consistent structure across projects
- AI agent behavior constrained by hooks and rules
- Decision history preserved in ADRs

**Negative:**
- Template convention overhead for trivial scripts
- Copier dependency for template updates
