# Architecture Decision Records

Key technical decisions for FRI_collect with context and tradeoffs.

## Format

Each ADR follows this template:

```markdown
# ADR-###: Title

**Status**: proposed | accepted | deprecated | superseded

**Context**: What problem are we solving?

**Decision**: What did we choose?

**Consequences**: What are the tradeoffs?
```

## Rules

- One ADR per significant technical decision
- Never delete old ADRs — mark as `deprecated` or `superseded`
- Reference ADRs in PRs that implement or change them
